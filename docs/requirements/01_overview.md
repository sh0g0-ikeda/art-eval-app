# 01. Overview（概要）

本書は、iOS/Android 向けのモバイルアプリとして提供する  
「模写評価AIアプリ（Drawing Evaluation App）」の要件定義書である。

本アプリは、運営側が用意した **「お題画像（課題画像）」** をユーザーに提示し、  
ユーザーがそのお題を模写して提出することで、

- サーバー側の画像モデルによる **定量的スコア評価**
- DeepSeek LLM による **定性的な講評・改善アドバイス**

を返す「AI模写コーチ」アプリである。

本章では、背景・目的・対象ユーザー・プロダクトコンセプト・使用技術の前提と、  
「お題ベース」の評価方式を含む全体像を示す。

---

## 1.1 背景（Background）

絵の練習、とくに「模写」の練習を行うユーザーは、次のような課題を抱えている。

- 自分の模写が **お手本にどれくらい近いのか** わからない  
- どこがズレているのか（比率、構図、線の精度など）を自力で分析しづらい  
- 上達しているかどうかが可視化されず、モチベーションが続きにくい  
- 指導者がいない環境では、具体的な講評や改善アドバイスが得られない  

また、ユーザーが自由に持ち込んだ任意の画像を評価しようとすると、

- 著作権・利用許諾の問題
- 画像条件（解像度・構図・光源など）のバラつき
- 元画像が手元にない／比較対象が不明

といった実務上の問題も発生する。

そこで本アプリでは、

- 運営側が事前に権利処理済みの **「お題画像（課題画像）」を用意**
- お題ごとに「正解画像（基準画像）」の特徴量や評価パラメータをサーバーに保持
- ユーザーはそのお題を模写し、スマホから撮影・投稿

という **「お題制・クエスト制の模写練習」** を採用する。

これにより、

- 評価ロジックの安定性・精度向上
- 著作権・権利面のクリア
- 難易度ごとのカリキュラム設計

が可能となる。

さらに、画像解析結果を DeepSeek LLM に入力することで、  
人間の講師に近い自然な講評を自動生成する。

---

## 1.2 目的（Purpose）

本アプリの目的は、以下の 3 点に整理される。

### ① お題ベースの模写を定量的に評価する

- 各お題画像に対して、模写画像との **類似度スコア（0〜100）** を算出  
- 構図のズレ、比率の誤差、線の精度などを **サブスコア** として可視化  
- 差分ヒートマップにより「どの部分がどれくらいズレているか」を直感的に表示  

### ② DeepSeek LLM による定性的な講評を提供する

- 定量評価の結果（スコア・サブスコア・ヒートマップ情報など）と  
  お題画像＋ユーザーの模写画像をもとに、DeepSeek API で講評文を生成する  
- 良い点・改善点・次の練習方針を、自然な日本語の文章で提示する  
- 人間の先生に添削してもらう感覚に近いフィードバック体験を目指す  

### ③ お題制のカリキュラムで継続的な学習を支援する

- 初級〜中級〜上級と、難易度ごとにお題を用意  
- お題ごとにスコアと講評が蓄積され、ユーザーごとの **上達履歴** を可視化  
- 「次はどのお題に挑戦すべきか」が分かるような学習導線を設計する

## 1.3 プロダクトの概要（Product Summary）

本アプリは、以下の 3 つのレイヤから構成される。

### 🔹 A. お題管理レイヤ（課題画像管理）

- 運営側が用意したお題画像（課題画像）をサーバー側で管理  
- お題ごとに以下の情報を保持する：
  - お題ID、タイトル、難易度、カテゴリ（人物・背景・静物など）
  - お題画像（基準画像）そのもの
  - 画像モデル用の事前計算済み特徴量
- モバイルアプリでは、
  - 「お題一覧」
  - 「お題詳細（基準画像の表示）」
  - 「このお題に模写を投稿する」フロー
  を提供する  

### 🔹 B. 定量評価レイヤ（画像解析）

- バックエンド（FastAPI + PyTorch）が、お題画像と模写画像の特徴量を比較  
- Cosine Similarity 等により類似度スコアを計算し、0〜100 のスコアに正規化  
- 構図のズレ・線の精度・比率の誤差などをサブスコアとして算出  
- OpenCV などを用いて差分ヒートマップを生成  

### 🔹 C. 定性的評価レイヤ（DeepSeek LLM）

- 定量評価の結果（スコア、サブスコア、差分情報など）と  
  お題の説明・難易度設定などを DeepSeek API に入力  
- LLM が、ユーザー向けの講評文を生成：
  - 良い点（例：線の安定、全体バランス など）
  - 改善点（例：目の位置が高め、輪郭がやや太い など）
  - 次の練習方針（例：線画練習を増やす、部分模写を行う 等）
- 結果画面で、スコア・ヒートマップ・講評文を一体として表示する

## 1.4 想定ユーザー（Target Users）

| 区分 | 内容 |
|------|------|
| 主対象ユーザー | 美術系学生、将来イラスト・漫画・ゲーム業界を目指すユーザー、趣味で模写練習を行う一般ユーザー |
| 副次対象ユーザー | 美術講師、教育機関、AI・画像処理分野の研究者 |
| 初期想定ユーザー規模（β版） | 約 500〜1,000 名 |

特に以下のようなニーズを持つユーザーを想定している。

- 体系的な模写練習をしたいが、独学で進めている  
- お題形式でチャレンジしながら、上達度を確認したい  
- SNS に上げる前に AI からフィードバックをもらいたい  

---

## 1.5 プロダクトコンセプト（Product Concept）

> **“お題を選んで描いて、AIに添削してもらう。  
>  定量スコア × LLM講評 × お題カリキュラムで上達を支援する。”**

- **お題制**：難易度やジャンルごとに用意された課題に取り組む  
- **定量評価**：スコアとヒートマップで、ズレを客観的に把握  
- **定性的評価**：DeepSeekによる講評で、次に何を直せばよいかが明確になる  
- **履歴管理**：お題ごとのスコア推移を見ながら、自分の成長を実感できる

## 1.6 想定使用環境・技術スタック（Supported Platforms & Tech Stack）

本アプリは、以下の技術スタックを前提とする。

### クライアント（モバイルアプリ）

- プラットフォーム：iOS / Android  
- 技術：React Native（＋必要に応じて Expo）  
- 言語：TypeScript / JavaScript  
- 主な役割：
  - お題一覧・お題詳細画面の表示
  - 模写画像の撮影・アップロード
  - 評価結果（スコア・ヒートマップ・講評）の表示
  - 評価履歴の一覧・詳細表示
  - ユーザー認証まわりの UI

### バックエンド（API サーバー）

- 技術：FastAPI（Python）  
- 主な役割：
  - お題画像・お題メタ情報の管理
  - 画像評価モデル（PyTorch）による定量評価
  - DeepSeek API の呼び出し（定性的評価）
  - ユーザー・認証・評価履歴の管理
  - ストレージ（S3 / Supabase Storage 等）との連携

### LLM・外部サービス

- LLM：DeepSeek API  
  - 定性的評価（講評生成）を担当  
- データベース：PostgreSQL など  
- ストレージ：AWS S3 もしくは Supabase Storage を想定

## 1.7 プロジェクト範囲（Scope）

### ✔ 初期リリース（β版）に含める範囲

- お題管理機能：
  - お題一覧の取得
  - お題詳細画面
  - お題ごとの模写投稿
- 定量評価機能：
  - 類似度スコア・サブスコア算出
  - 差分ヒートマップ生成
- 定性的評価機能：
  - DeepSeek API による講評生成
- ユーザー管理機能：
  - ログイン/ログアウト（Google / Email など）
  - ユーザーごとの評価履歴管理
- 評価結果表示機能：
  - スコア・ヒートマップ・講評の統合表示
- シンプルな共有機能：
  - 結果画面のシェア（画像・テキストベース）

### ✘ 初期リリースで範囲外（将来拡張）

- ユーザーが任意画像をアップロードして評価するモード  
- お題画像のユーザー投稿・審査・公開機能  
- AI によるお手本画像の自動生成機能  
- アプリ内コミュニティ（コメント、いいね、ランキング等）  
- オンデバイス推論によるオフライン評価
