# 09. UI / UX 要件（UI / UX Requirements）

本章では、Drawing Evaluation App の画面構成および UX 方針を定義する。  
Chapter 03（Functional Requirements）および Chapter 06（API Spec）、Chapter 07（Evaluation Logic）で定義した機能・評価ロジックを、ユーザーにとって分かりやすく・継続利用しやすい形で提供することを目的とする。

---

## 9.1 全体コンセプト

- ターゲット：美術系学生・趣味で絵を描くユーザー
- コンセプト：  
  - 「AI 模写コーチ」  
  - スコアよりも **学びの実感** を重視  
  - 白黒線画ベースで、シンプルかつ落ち着いたトーン

### 9.1.1 デザイン方針

- 色数を抑えたモノトーンベース（白・黒・グレー＋アクセントカラー1色）
- 余白を広めに取り、イラスト画像が主役になるレイアウト
- 評価結果画面では、「スコア」「ヒートマップ」「講評」を一画面に集約
- アニメーションは最小限（ローディングや画面遷移のみに限定）

---

## 9.2 画面一覧

本アプリで提供する主要画面は以下の通り。

| ID | 画面名 | 概要 |
|----|--------|------|
| S-01 | スプラッシュ / 起動中画面 | アプリ起動時に表示 |
| S-02 | ログイン画面 | Googleログイン |
| S-03 | ホーム / お題一覧画面 | お題カードの一覧表示 |
| S-04 | お題詳細画面 | お題画像＋説明＋「模写を投稿」ボタン |
| S-05 | 模写投稿画面 | カメラ/ギャラリーから選択 → 送信 |
| S-06 | 評価進行中画面 | 評価処理中のローディング表示 |
| S-07 | 評価結果画面 | スコア＆ヒートマップ＆講評 |
| S-08 | 履歴一覧画面 | 過去提出の一覧 |
| S-09 | 履歴詳細画面 | 過去の評価結果を再表示 |
| S-10 | 設定 / プロフィール画面 | 表示名・アイコンなど（MVPでは簡易） |

---

## 9.3 画面別要件

### 9.3.1 S-01 スプラッシュ / 起動中画面

**目的**  
- アプリ起動時の初期処理（トークン確認等）中の待ち時間をカバーする。

**表示要素**

- アプリロゴ（テキストでも可）
- サブコピー（例：「AI 模写コーチ」）
- ローディングインジケータ（スピナー）

**挙動**

- 起動時に最大数秒表示
- ログイン済みなら S-03（ホーム）へ遷移
- 未ログインなら S-02（ログイン画面）へ遷移

---

### 9.3.2 S-02 ログイン画面

**目的**  
- ユーザーに Google ログインを促す。

**表示要素**

- シンプルなロゴ／アプリ名
- 「Google でログイン」ボタン（公式ガイドラインに準拠）
- プライバシーポリシー／利用規約へのリンク（将来）

**UX要件**

- 初回起動時は必ずこの画面へ
- ログイン済みの場合は、次回以降自動スキップ
- ログイン失敗時はわかりやすいメッセージを表示

---

### 9.3.3 S-03 ホーム / お題一覧画面

**目的**  
- ユーザーに対して「どのお題を模写するか」を選ばせる。

**表示要素**

- 上部：簡単な挨拶 or 今日のヒント（例：「今日は輪郭を意識して描いてみましょう」）
- お題カード一覧（縦スクロール）
  - サムネイル画像（白黒）
  - タイトル
  - 難易度（beginner / intermediate / advanced）
  - カテゴリ（アイコン＋ラベル）

**フィルタ / ソート（将来的に追加可）**

- 難易度フィルタ（ドロップダウン）
- カテゴリフィルタ
- 並び替え（人気順 / 新着順）

**アクション**

- カードタップ → S-04（お題詳細）へ遷移
- 下部タブ or メニューから「履歴」（S-08）への遷移

---

### 9.3.4 S-04 お題詳細画面

**目的**  
- お題の内容を理解させ、「模写を投稿する」アクションへ導く。

**表示要素**

- お題画像（白黒。ズーム可能だと望ましい）
- タイトル
- 難易度・カテゴリ
- 説明文（description）
- コツ（tips）

**アクション**

- 「このお題を模写する」ボタン
  - 押下 → S-05（模写投稿画面）へ

---

### 9.3.5 S-05 模写投稿画面

**目的**  
- ユーザーが描いた模写をアップロードさせる。

**表示要素**

- お題のサムネイル（小さく表示）
- 「カメラで撮影」「ギャラリーから選択」ボタン
- 選択後のプレビュー（白黒化はサーバー側だが、プレビューはカラーでも可）

**アクションフロー**

1. ユーザーが画像を選択・撮影
2. プレビュー表示
3. 「この画像で評価する」ボタン押下
4. `/challenges/{id}/submissions` にアップロード
5. 送信完了後 S-06（評価進行中）へ遷移

**UX要件**

- 送信中は重複タップを防ぐ（ボタンを無効化）
- 通信エラー時は再送信ボタンを表示

---

### 9.3.6 S-06 評価進行中画面

**目的**  
- モデル処理・DeepSeek 講評生成の待ち時間をストレスなく過ごさせる。

**表示要素**

- 簡単なメッセージ（例：「AIがあなたの模写をチェックしています…」）
- ローディングインジケータ
- 進捗イメージ（ステップ表示など：前処理 → スコア計算 → 講評生成）

**挙動**

- バックエンド上では `/submissions/{submission_id}` を一定間隔でポーリング
- `status = completed` になったタイミングで S-07 に自動遷移
- `failed` の場合はエラーメッセージを表示し、S-05へ戻る導線を用意

---

### 9.3.7 S-07 評価結果画面（コア画面）

**目的**  
- ユーザーに自分の模写を客観的に理解させ、改善の意欲を高める。

**表示要素**

1. ヘッダー部
   - お題タイトル
   - 提出日時
2. スコア部
   - 総合スコア（0〜100）：大きく表示
   - サブスコア（構図 / 線精度 / 比率）：棒グラフ or ラベル
3. ヒートマップ部
   - お題画像と、差分ヒートマップ画像（横並び、スワイプで切替など）
4. 講評部（DeepSeek）
   - summary：1〜2文の要約
   - strengths（良い点）：箇条書き
   - weaknesses（改善余地）：箇条書き
   - suggestions（具体的な次の一手）：箇条書き

**UX要件**

- スコアだけでなく「何が良かったか」「何を直せばいいか」が一目で分かる構成
- 講評はなるべく読みやすく、改行・箇条書きを使う
- 下部に「もう一度描く」「他のお題を探す」「履歴を見る」などのボタンを設置

---

### 9.3.8 S-08 履歴一覧画面

**目的**  
- 自分の成長を俯瞰して見られるようにする。

**表示要素**

- 日付順（新しい順）に並んだカードリスト
  - お題サムネイル
  - お題タイトル
  - スコア（overall_score）
  - 提出日時

**アクション**

- カードタップ → S-09（履歴詳細画面）へ遷移

---

### 9.3.9 S-09 履歴詳細画面

**目的**  
- 過去の評価結果を再度確認し、「前回より伸びたか」を意識させる。

**表示要素**

- S-07（評価結果画面）とほぼ同様
- 必要があれば「同じお題の最新スコアとの比較」を簡易表示
  - 例：「前回より +5 点アップ！」

---

### 9.3.10 S-10 設定 / プロフィール画面（MVP版）

**目的**  
- 最低限のアカウント情報を確認できるようにする。

**表示要素**

- ユーザー名
- アイコン（変更はMVPでは任意）
- ログアウトボタン

---

## 9.4 ナビゲーション設計

### 9.4.1 グローバルナビゲーション

MVP では、シンプルなボトムタブ or 上部タブを想定する。

- ホーム（お題一覧）  
- 履歴  
- 設定  

※ 画面数が少ないため、初期は「ホーム左上のメニューボタン → 履歴・設定」といった簡易ナビでも可。

---

## 9.5 UX ポイント（心理的デザイン）

- スコアが低くてもモチベーションが落ちないようにする
  - 講評では「良い点」を必ず最初に表示
  - 文言は「ダメです」ではなく「ここを直すとさらに良くなる」系
- 何度も同じお題をやり直したくなるような導線
  - 評価結果画面に「同じお題でもう一度描く」ボタン
- ユーザーの連続利用を促す
  - （将来）「連続日数」バッジ
  - （将来）「目標スコア」の設定

---

## 9.6 アクセシビリティ・対応方針（簡易）

- 文字サイズは OS のアクセシビリティ設定に追従可能な設計を目指す
- 色覚多様性に配慮し、ヒートマップの配色は検討（必要に応じて切替）
- コントラスト比が極端に低くならないような配色
